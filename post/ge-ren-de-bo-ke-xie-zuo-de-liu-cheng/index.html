浅谈个人的博客写作的流程

2020-11-13 3 min read # 博客 # 随笔

简单记录和聊一聊一下本人的写作流程。关于写一篇博客文章的想法产生与最后的产出。

定题与选择

定题通常是在在我浏览一些文章[1]，或者在生活中有什么感触时头脑中会有一个大致想法，通常灵感来源是前一种（浏览我的博客你会发现关于现实生活的文章很少）。然后我的习惯是不写到纸上，而是直接打开电脑，如今我的文本编辑器不是什么 Markdown 编辑器，而是 Visual Code，加上一个支持 Markdown 的扩展，毫不夸张地说，VS 已经相当一个公文包了，啥都能做（对我而言）。

大纲与思维导图

然后以我个人比较随意的性格，可能在选择素材或者细方向这个步骤上，是与写作平行的，是的，我习惯边写边收集素材。通常都是文本编辑器+浏览器的形式。
当然现在我可能会在写的时候写个大纲，也就是大标题的形式而已，算是认识到大纲的重要性和写作要有起码的规范这两条“规矩”吧,整理出来的大纲/导图也可以作为一些攻略文章的配图直接贴出来，方便读者看清思路和流程。生成精美思维导图、流程图的网站比比皆是，所以不用我介绍了。

一口气写完吧

我的写作流最大的特点可能就是，要么不写，要么一口气写完（大部分）。马不停蹄，马拉松式写作。有时候，别人写一个攻略啥的都是先做了一边然后再写成文章，我通常是边写边做，写着写着顺便把问题解决了（或者我被问题解决，然后放弃谋篇文章再也不写）。这时候很需要注意自己的写作环境，好的写作环境会让你有更大的写作动力与效率。桌椅是否合适？你的写作设备是否合手？写作工具的颜值/设计让你有写作动力吗？等等问题。
在写作过程中的检查也是必要的一环，我通常是边写边检查，写完再检查一遍。

文章配图与封面

配图我也是边写边截图/制作然后直接上传到图床，因为我认为如果在写作过程中直接插入配图（不是那种图文无关的图），可以让文章写时看起来更完整些，方便调整。封面是我写完后若是文章主题范围大就在无版权网站上（比如 Unsplash[2] ）找一张，若是能在能力范围内就自己制作封面，这也是我今后的打算。

检查、检查、还是检查

写完后检查排版、格式是否正确显示，检查错字漏字，检查语法错误，可以用一些语法检查网站辅助检查一下。有些工具可以看看这篇文章[3]，很多工具也是在我刷的网站里可以搜索到的。
